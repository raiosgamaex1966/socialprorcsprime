Introdução

O Social Pro é uma plataforma de rede social full-stack, pronta para produção, construída com React 18, TypeScript 5, Vite 5, Tailwind CSS v3 e alimentada pelo Supabase (PostgreSQL, Auth, Storage, Edge Functions, Realtime). Projetada e mantida por afCode.

👥
Rede Social

Publicações, stories, reels, amigos, grupos, páginas, eventos e mensagens em tempo real.

🛒
Marketplace

Compra e venda com anúncios, ofertas, avaliações de vendedores, vendedores verificados e detecção de fraudes.

🎬
Watch e Reels

Plataforma de vídeo com playlists, indicadores de transmissão ao vivo, curtidas e comentários.

🛡️
Admin e Moderação

Painel administrativo completo com gerenciamento de usuários, moderação de conteúdo, CMS e design do site.

💰
Créditos e Publicidade

Economia de créditos com recarga, presentes, posts patrocinados e impulsionamento de páginas/posts/anúncios.

🔐
Segurança Corporativa

Row-Level Security (RLS) em mais de 50 tabelas, funções RBAC, denúncias de conteúdo, avisos e banimentos de usuários.

📖 Como ler esta documentação Siga as seções 1 a 6 em ordem para uma nova instalação. Se você estiver atualizando uma instalação existente, pule para Atualização e Migração. Para detalhes sobre os recursos, consulte Recursos Principais e Monetização.

2. Instalação e Configuração

Siga estes passos na ordem exata para uma nova instalação:

  1. Clonar o Repositório
    Terminal
    git clone https://github.com/your-org/Social Pro.git
cd Social Pro
  2. Instalar Dependências
    Terminal
    # Recomendado (rápido)
bun install

# Alternativa
npm install

    Isso instala cerca de 80 pacotes, incluindo React, Tailwind, Supabase SDK, Leaflet, Recharts e componentes do shadcn/ui.

  3. Configurar Variáveis de Ambiente

    Consulte a Etapa 3: Variáveis de Ambiente abaixo para ver a lista completa.

  4. Executar Migrações do Banco de Dados

    Se estiver hospedando por conta própria com seu próprio Supabase, aplique todas as migrações da pasta supabase/migrations/:

    Terminal
    # Vincule o seu projeto Supabase
supabase link --project-ref YOUR_PROJECT_REF

# Envie todas as migrações
supabase db push
    ℹ️ Usuários do afCode Cloud Se você compilou este projeto na plataforma afCode, as migrações são aplicadas automaticamente. Pule esta etapa.
  5. Iniciar o Servidor de Desenvolvimento
    Terminal
    bun dev    # ou: npm run dev

    Abre em http://localhost:8080 (servidor de desenvolvimento Vite com HMR).

  6. Executar o Assistente de Primeira Execução

    No primeiro acesso, você será redirecionado para /setup. Consulte a Etapa 4.

✅ Resumo do Início Rápido git clone → bun install → configure .env → supabase db push → bun dev → /setup

3. Variáveis de Ambiente

Crie um arquivo .env na raiz do projeto. As variáveis prefixadas com VITE_ são expostas ao bundle do cliente.

Variáveis Obrigatórias

Variável Status Descrição Exemplo
VITE_SUPABASE_URL Obrigatório URL da API do seu projeto Supabase https://xxxx.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY Obrigatório Chave pública/anônima do Supabase (segura para o cliente) eyJhbGci…
VITE_SUPABASE_PROJECT_ID Obrigatório ID de referência do projeto (usado para URLs de Edge Functions) abcdefghijklmnop

Variáveis Opcionais

Variável Status Descrição Padrão
VITE_DEMO_MODE Opcional Ativar modo de demonstração/teste com painel de login rápido (consulte o Modo Demo) false

Segredos de Backend (Supabase Edge Functions)

Estes são definidos no painel do seu projeto Supabase (ou afCode Cloud → Segredos), e não no arquivo .env:

Segredo Objetivo
SUPABASE_SERVICE_ROLE_KEY Acesso à API de nível de administrador para Edge Functions (setup-admin, etc.)
LOVABLE_API_KEY Recursos de IA (tradução, etc.) — definidos automaticamente no afCode Cloud

Exemplo de Arquivo .env

.env
VITE_SUPABASE_URL="https://your-project.supabase.co"
VITE_SUPABASE_PUBLISHABLE_KEY="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
VITE_SUPABASE_PROJECT_ID="your-project-ref"

# Opcional: Ativa o modo demo (mostra o painel de login rápido na página de autenticação)
# Defina como "true" para homologação/demo, remova ou defina como "false" para produção
VITE_DEMO_MODE=false
⚠️ Segurança Nunca envie o arquivo .env para o Git. O arquivo .gitignore já o exclui. Apenas os valores prefixados com VITE_ chegam ao navegador — nunca coloque chaves de service role em variáveis VITE_.

4. Assistente de Primeira Execução

O Assistente de Configuração é executado automaticamente na primeira visita quando não existe nenhum Super Admin. Ele cria a sua conta de administrador e configura a plataforma.

Redirecionamento Automático

Condição Comportamento
Nenhum admin na tabela user_roles Todas as rotas redirecionam para /setup
Admin existe, usuário não está logado Rotas protegidas redirecionam para /auth
Admin existe, usuário logado Acesso normal ao aplicativo com base na função

Etapas do Assistente

  1. Passo 1 — Criar Super Admin

    Insira o nome de exibição, e-mail e senha (mínimo de 8 caracteres, confirmada). A conta é criada por meio de uma Edge Function setup-admin segura com e-mail confirmado automaticamente e a função admin atribuída no servidor.

  2. Passo 2 — Identidade do App

    Configure o nome do site, descrição, URL e envie a identidade visual (logo claro, logo escuro, favicon). Os arquivos são armazenados no bucket site-assets.

  3. Passo 3 — Configurações do Sistema

    Alterne o registro de usuários, verificação de e-mail, modo de manutenção e selecione o idioma padrão.

  4. Passo 4 — Revisar e Iniciar

    Resumo de todas as configurações. Clique em "Iniciar Aplicativo" → define o sinalizador setup_complete → redireciona para a página inicial.

🚨 Apenas uma vez Assim que a configuração for concluída, a rota /setup redirecionará para /auth. A Edge Function impede a criação duplicada de admins. Para executar o assistente novamente, exclua a linha do administrador em user_roles e a configuração setup_complete em site_settings.

Arquivo _redirects do Netlify

public/_redirects
/*    /index.html   200

Configuração do Nginx (VPS)

nginx.conf
server {
    listen 80;
    server_name yourdomain.com;
    root /var/www/Social Pro/dist;
    index index.html;

    # Fallback de SPA — crítico para roteamento no cliente
    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache agressivo para assets estáticos
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Cabeçalhos de segurança
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
}

Implantação com Docker

Dockerfile
# Etapa 1: Build
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json bun.lockb ./
RUN npm install
COPY . .
RUN npm run build

# Etapa 2: Servir
FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Atualização e Migração — Visão Geral

Quando uma nova versão do Social Pro é lançada, você precisa atualizar três camadas:

🖥️
1. Frontend

Obtenha o código mais recente, instale as dependências, recompile e implante novamente o pacote estático.

🗄️
2. Banco de Dados

Aplique novas migrações SQL para adicionar tabelas, colunas, políticas de segurança (RLS) e funções.

⚡
3. Edge Functions

Implante funções serverless novas ou atualizadas.

⚠️ Sempre faça backup primeiro Antes de atualizar, exporte seu banco de dados e faça uma cópia da sua pasta dist/ atual. Consulte o Procedimento de Reversão.

Atualização do Frontend

Para Projetos no afCode Cloud

As atualizações do frontend são automáticas quando você clica em Publish → Update. Nenhuma ação manual é necessária.

Para Projetos Hospedados por Conta Própria / Sincronizados com o GitHub

  1. Obter o Código Mais Recente
    Terminal
    cd Social Pro
git pull origin main

# Se estiver usando o download em ZIP: extraia a nova versão
# Mantenha seu arquivo .env — não o substitua
  2. Instalar Dependências Atualizadas
    Terminal
    rm -rf node_modules
bun install   # or: npm install
  3. Verificar Novas Variáveis de Ambiente

    Compare seu .env com o registro de alterações ou com a seção de Variáveis de Ambiente.

  4. Compilar e Testar Localmente
    Terminal
    npx tsc --noEmit   # Verifique a tipagem primeiro
bun run build      # Build de produção
bun run preview    # Pré-visualizar localmente
  5. Implantar

    Envie a nova pasta dist/ para o seu provedor de hospedagem (consulte a seção de Implantação).

Atualização do Banco de Dados e Backend

Cada versão pode incluir novas migrações SQL em supabase/migrations/.

Para Projetos no afCode Cloud

As migrações do banco de dados são aplicadas automaticamente. Nenhuma etapa manual é necessária.

Para Supabase Hospedado por Conta Própria

  1. Revisar Novas Migrações
    Terminal
    ls -la supabase/migrations/
cat supabase/migrations/YYYYMMDDHHMMSS_migration_name.sql
  2. Aplicar Migrações
    Terminal — Opção A: Supabase CLI
    supabase link --project-ref YOUR_PROJECT_REF
supabase db push
    Terminal — Opção B: SQL Manual
    psql "postgresql://postgres:YOUR_PASSWORD@db.YOUR_REF.supabase.co:5432/postgres"
\i supabase/migrations/YYYYMMDDHHMMSS_new_migration.sql
  3. Verificar Esquema
    SQL
    SELECT tablename FROM pg_tables WHERE schemaname = 'public' ORDER BY tablename;
SELECT tablename, rowsecurity FROM pg_tables WHERE schemaname = 'public' AND rowsecurity = true;
🚨 A Ordem das Migrações é Importante Sempre aplique as migrações em ordem cronológica (ordenadas pelo carimbo de data/hora do nome do arquivo). Nunca pule uma migração.

Atualização de Edge Functions

Para afCode Cloud

As Edge Functions são implantadas automaticamente quando o código é alterado. Nenhuma ação é necessária.

Para Supabase Hospedado por Conta Própria

Terminal
# Implanta todas as Edge Functions de uma vez
supabase functions deploy --all

# Ou implante individualmente
supabase functions deploy setup-admin
supabase functions deploy translate-post
supabase functions deploy embed-post
supabase functions deploy tenor-proxy

Verifique se cada função responde (deve retornar 400 ou 200, não 404):

Terminal
curl -i https://YOUR_REF.supabase.co/functions/v1/setup-admin \
  -H "Authorization: Bearer YOUR_ANON_KEY"

Buckets de Armazenamento

Novas versões podem exigir buckets de armazenamento adicionais. Verifique se todos os 10 buckets existem:

Bucket Público Objetivo

post-images ✅ Anexos de imagens de posts e imagens de posts patrocinados
post-videos ✅ Anexos de vídeo de posts
story-media ✅ Fotos e vídeos de Stories
reel-videos ✅ Conteúdo de vídeo de Reels

group-images ✅ Avatares e capas de grupos

SQL — Criar buckets ausentes 
INSERT INTO storage.buckets (id, name, public)
VALUES ('site-assets', 'site-assets', true)
ON CONFLICT (id) DO NOTHING;

Procedimento de Reversão

Reversão do Frontend

Terminal
rm -rf dist/
cp -r dist-backup/ dist/

# Ou reverta via git
git checkout HEAD~1
bun install && bun run build

Reversão do Banco de Dados

🚨 Reversões de banco de dados são destrutivas As migrações SQL não podem ser revertidas automaticamente. Restaure a partir do seu backup:
Terminal
psql "postgresql://postgres:PASSWORD@db.REF.supabase.co:5432/postgres" < backup.sql

Projetos no afCode Cloud

Use o recurso nativo de Histórico para restaurar uma versão anterior. Clique no ícone do relógio no editor para navegar e restaurar estados anteriores.

Autenticação

Métodos de Login

  • E-mail e Senha — Login padrão com verificação opcional de e-mail
  • Google OAuth — Login com um clique via conta do Google
  • Apple Sign-In — Login com ID Apple

Fluxo de Registro em 7 Etapas

  1. Detalhes da conta (nome, e-mail, senha com indicador de força)
  2. Informações pessoais (data de nascimento, gênero, telefone, localização)
  3. Envio de foto de perfil com ferramenta de corte (crop)
  4. Envio de foto de capa
  5. Seção Biografia / Sobre
  6. Seleção de interesses (escolha até 8 de 14 categorias)
  7. Resumo de busca de amigos e envio

Redefinição de Senha

Os usuários clicam em "Esqueci a Senha" → recebem o e-mail de redefinição → são redirecionados para /reset-password → definem a nova senha.

Verificação de E-mail

Configurável nas Configurações do Admin. Quando ativado, os usuários devem verificar o e-mail antes de acessar o aplicativo. Inclui botão de reenvio com tempo de espera de 60 segundos.

Login Rápido no Modo Demo

Quando VITE_DEMO_MODE=true, um painel de "Login Rápido de Demonstração" aparece na página de autenticação com três contas baseadas em funções. Consulte o Modo de Demonstração / Teste para obter detalhes.

Perfis de Usuário

Armazenado na tabela profiles, criado automaticamente no cadastro por meio de uma trigger no banco de dados.

Dados do Perfil

  • Nome de exibição, biografia, localização, site
  • Avatar e foto de capa com suporte a corte (armazenados no bucket profile-images)
  • Data de nascimento, gênero, telefone, educação, local de trabalho
  • Interesses (array), status de relacionamento
  • Presença online (last_seen_at com rastreamento em tempo real)
  • Barra de progresso de conclusão do perfil

Abas do Perfil

  • Publicações — Linha do tempo do usuário
  • Sobre — Cartão de introdução com detalhes pessoais
  • Amigos — Lista de amigos com gerenciamento
  • Fotos — Galeria de fotos com lightbox
  • Check-ins — Check-ins de localização com mapa (Leaflet)

Publicações e Feed

  • Carrossel de várias imagens, envios de vídeo, estilos de fundo, sentimentos, localização de check-in
  • Privacidade: Público, Amigos, Somente Eu
  • 6 tipos de reações (curtir, amei, haha, uau, triste, grrrr), comentários em tópicos
  • Compartilhar, salvar/favoritar, fixar no perfil, arquivar
  • Agendamento de publicações (visualizações de calendário e semanal)
  • Histórico de edições com visualização de diferenças (diff), incorporação de publicações, tradução por IA
  • Pré-visualização de links com metadados OpenGraph
  • Ocultar publicações, denunciar publicações, desativar comentários por publicação
  • Impulsionar publicações com créditos de promoção

Stories e Reels

Stories

  • Stories de foto/vídeo, expiração automática em 24 horas
  • Visualizador em tela cheia com indicadores de progresso
  • Reações (emoji + mensagem opcional)
  • Destaques do Story com capas personalizadas
  • Armazenamento: bucket story-media (máximo de 50MB)

Reels

  • Vídeo curto (máximo de 100MB), envio com legenda e miniatura
  • Curtidas, comentários em tópicos, compartilhamento, contagem de visualizações
  • Sistema de seguir criadores
  • Armazenamento: bucket reel-videos

Mensagens

Chat 1:1 e em grupo em tempo real com mensagens completas:

  • Mensagens diretas e chats em grupo (funções de administrador)
  • Anexos de arquivos, imagens e mensagens de voz (máximo de 10MB)
  • Reações de mensagem (emoji), recibos de leitura, indicadores de digitação
  • Fixar, editar, excluir e encaminhar mensagens
  • Seletor de GIFs (proxy do Tenor), seletor de emojis
  • Configurações do chat em grupo (renomear, avatar, adicionar/remover membros)

Grupos

  • Grupos públicos/privados com 10 categorias
  • Publicações do grupo com suporte a fixação, comentários, reações
  • Conversa de grupo criada automaticamente (chat vinculado ao grupo)
  • Eventos com RSVP (no escopo do grupo)
  • Funções: admin, moderador, membro
  • Links de convite compartilháveis com expiração e limites de uso
  • Preferências de notificação por grupo (silenciar, alternar por tipo)
  • Análise (crescimento de membros, engajamento), feed de atividades, regras
  • Hover cards para visualização rápida
  • Impulsionar publicações do grupo com créditos de promoção

Páginas

  • Páginas de empresas/organizações com categorias e URLs amigáveis (slug)
  • Publicações de página com agendamento, comentários, reações
  • Sistema de seguidores, suporte a múltiplos administradores
  • Eventos (no escopo da página), análise, hover cards
  • Informações de contato: e-mail, telefone, site, localização
  • Impulsionar páginas inteiras e posts de páginas individuais com créditos

Eventos

  • 10 categorias: Social, Negócios, Educação, Entretenimento, Esportes, Tecnologia, Saúde, Artes, Comunidade, Outro
  • RSVP: Confirmado, Interessado, Não Vou com lista de participantes
  • Recorrência: Nenhuma, Diariamente, Semanalmente, Mensalmente, Anualmente com data de término
  • Lembretes: Configuráveis (15 minutos a 1 semana antes)
  • Visualização em calendário (mensal com pontos) e visualização em mapa (agrupamento do Leaflet)
  • Comentários em tópicos, eventos relacionados, exportação .ics, modal de compartilhamento
  • Resumo semanal por e-mail via Edge Function

Marketplace

Anúncios

  • 10 categorias, 4 condições, múltiplas imagens, localização
  • Status: Ativo, Vendido, Reservado
  • Modelos para criação rápida, gráficos de histórico de preços
  • Contagem de visualizações, salvar/favoritar, visualizados recentemente

Transações e Confiança

  • Sistema de ofertas (comprador → vendedor com mensagem opcional)
  • Painel do vendedor com análises
  • Avaliações e classificações de vendedores, selos de vendedor verificado
  • Detecção automatizada de fraudes (Edge Function)
  • Anúncios promovidos baseados em créditos com segmentação
  • Denúncias de anúncios com fila de moderação

Watch

  • Envios de vídeo com 13 categorias
  • Playlists (públicas/privadas), indicador de transmissão ao vivo
  • Curtidas, comentários em tópicos, salvar, rastreamento de visualizações
  • Recomendações por categoria

Notificações

  • Notificações em tempo real no aplicativo com indicador de não lidas
  • Tipos: solicitações de amizade, curtidas, comentários, compartilhamentos, convites de grupo, lembretes de eventos, mensagens
  • Notificações de crédito: presente recebido, presente do admin, confirmação de gastos, aviso de saldo baixo

Sistema de Créditos

O Social Pro inclui uma economia de créditos de promoção completa para monetizar conteúdos impulsionados e publicidade.

Saldo de Créditos

  • Cada usuário possui um registro em promotion_credits com um campo balance
  • Saldo exibido na página de Créditos (/credits) e na Central de Anúncios (/advertising)
  • Histórico de transações na tabela credit_transactions (tipo: spend, topup, gift, admin_gift)

Métodos de Recarga

Método Descrição
Pacotes de Crédito Os usuários compram pacotes (50, 100, 250, 500, 1000 créditos) por meio da página de Créditos

Presente de Usuário para Usuário Os usuários podem enviar créditos como presente para amigos por meio do modal de Enviar Créditos

Impulsionamentos e Promoções

Os usuários podem impulsionar vários tipos de conteúdo usando seus créditos de promoção:

Conteúdo Impulsionável

Conteúdo Localização Opções de Duração Custo em Créditos
Publicações do Feed Menu de ação do post → "Impulsionar publicação" 3, 7, 14 dias 15, 30, 50 créditos
Páginas Cabeçalho da página → Botão "Impulsionar Página" (apenas proprietário) 3, 7, 14 dias 20, 40, 70 créditos
Publicações de Página Post da página → Ícone de foguete (apenas proprietário) 3, 7, 14 dias 15, 30, 50 créditos
Publicações de Grupo Menu suspenso do post do grupo → "Impulsionar publicação" (apenas autor) 3, 7, 14 dias 15, 30, 50 créditos
Anúncios do Marketplace Anúncio → Botão "Promover" 3, 7, 14 dias 10, 20, 35 créditos
Publicações Patrocinadas Publicidade → "Criar Anúncio" 1, 3, 7 dias 10, 25, 50 créditos

Como Funciona o Impulsionamento

  1. O usuário clica em "Impulsionar" em um conteúdo compatível
  2. O modal exibe as opções de nível com os custos em créditos
  3. Os créditos são deduzidos da tabela promotion_credits
  4. A transação é registrada em credit_transactions
  5. O post patrocinado é criado na tabela sponsored_posts com as datas de início e término
  6. O conteúdo aparece nos feeds com o selo "Patrocinado" durante o período ativo
  7. Aviso de saldo baixo se os créditos restantes forem ≤10

Resolução de Problemas (Troubleshooting)

O aplicativo redireciona para /setup repetidamente

A tabela user_roles está vazia ou sua política RLS bloqueia leituras. Certifique-se de que existe pelo menos uma linha de administrador.

O assistente de configuração falha ao criar o administrador

A Edge Function setup-admin requer a chave SUPABASE_SERVICE_ROLE_KEY como segredo. Verifique se ela está configurada em seus segredos de backend.

Imagens/Logos não carregam

Verifique se o bucket site-assets existe e é público. Verifique as URLs em site_settings → admin_design.

Tempo real (Realtime) não está funcionando

Verifique se as tabelas foram adicionadas à publicação em tempo real: ALTER PUBLICATION supabase_realtime ADD TABLE messages;

Permissão RLS negada

O usuário pode não estar autenticado ou não ter a função necessária. Verifique as políticas usando a função has_role().

A compilação (build) falha com erros de TypeScript

Terminal
npx tsc --noEmit           # Visualizar os erros exatos
rm -rf node_modules/.vite  # Limpar cache do Vite
bun install && bun run build

Login demo não aparece

Verifique se VITE_DEMO_MODE=true está no seu arquivo .env (e não DEMO_MODE). Reinicie o servidor de desenvolvimento após alterar as variáveis de ambiente. Certifique-se de que as contas demo existem no Auth.

Edge Function retorna 404

A função pode não ter sido implantada. Execute supabase functions deploy nome-da-funcao ou faça a implantação novamente pelo seu projeto no afCode Cloud.

Os créditos não estão sendo deduzidos / o saldo não muda

Verifique se o usuário tem uma linha em promotion_credits. Se não, insira uma com balance: 0. Verifique se as políticas RLS permitem que o usuário atualize seu próprio saldo de créditos.

Post patrocinado não aparece no feed

Verifique se is_active = true, start_date ≤ agora e end_date ≥ agora. Além disso, verifique se o limite de frequência não foi excedido para o usuário que está visualizando.

Perguntas Frequentes (FAQ)

Como atualizo para uma nova versão?

Siga o guia de Atualização e Migração. Em resumo: obter o código mais recente → instalar dependências → aplicar migrações → implantar Edge Functions → compilar → implantar frontend.

Posso usar meu próprio projeto Supabase?

Sim. Defina a URL e as chaves do seu Supabase no arquivo .env, aplique todas as migrações da pasta supabase/migrations/, crie os buckets de armazenamento e implante as Edge Functions.

Como adiciono um novo administrador?

Um administrador existente pode atribuir a função administrativa acessando Painel Administrativo → Gerenciamento de Usuários → clique em um usuário → altere a função para admin.

O aplicativo está pronto para celulares (mobile-ready)?

Sim. O Social Pro é totalmente responsivo com um design mobile-first, barra lateral adaptativa, gestos de toque e notificações dispensáveis deslizando o dedo.

Posso personalizar a identidade visual?

Sim. Acesse Painel Administrativo → Design do Site para enviar os logos, favicon e definir as cores do tema. Também é possível injetar CSS/JS personalizado.

Como ativo/desativo recursos?

Painel Administrativo → Configurações → aba Recursos. Ative/desative recursos específicos da plataforma.

Onde os dados dos usuários são armazenados?

Banco de dados PostgreSQL via Supabase. Arquivos nos buckets do Supabase Storage. Todos os dados são protegidos por políticas de Row-Level Security (RLS).

Como dou créditos aos usuários?

Os administradores podem doar créditos acessando Painel Administrativo → Gerenciamento de Usuários → selecionar usuário → Doar Créditos. Os usuários também podem comprar pacotes de créditos em /credits ou receber presentes de outros usuários.

Como gerencio anúncios como administrador?

Painel Administrativo → Anúncios permite gerenciar 6 espaços de anúncios HTML (cabeçalho, barra lateral, rodapé, 3 no feed). Os usuários autogerenciam seus posts patrocinados a partir de /advertising.

Como funciona o modo demo?

Defina VITE_DEMO_MODE=true em .env para exibir um painel de login rápido com 3 contas pré-configuradas (User, Moderator, Admin). Remova ou defina como false para produção. Consulte o Modo Demo.
